Settings ウィンドウ

 

   Settings ウィンドウは、WebLEAPで使用するサーチエンジンを選んだり、それぞれのサーチエンジンの設定を行ったりするウィンドウです。例文の検索範囲をドメイン名で絞ったりすることもできます(Google Web API 使用時のみ)。

 

 

       “Search Engine”の右のボタンで、使用するサーチエンジンを選びます。

       “search options(for google)” の右のボタンは、検索範囲をドメイン名で絞ったりするときに使います。

       “GoogleAIP Licence Key(for google) の右の欄に、Google API ライセンスキーを入力します。Google を利用する場合は、まず、　Google Web API のライセンスキーを取得した後、これをこの欄に書き込み、設定を保存してください。

       “proxy” の右は、インターネットにアクセスする際に利用する　proxy の設定です。”use the Proxy”チェックボタンは、proxy を使用するか否かを設定します。その右は、使用するproxy のホスト名とポート番号です。

 

       右上の[exit]ボタンは、このウィンドウを閉じます。

       右上の[save]ボタンは、このウィンドウの設定内容を保存します。設定内容は、dsr/bin/webleap.cfg に保存されています。このファイルを削除すると、初期状態に戻ります。なお、dsr/bin/webleap.cfg がユーザによって書き込めるようになっておく必要があります。

 

       WebLEAP は、与えられた文章を単語列に分解し、それぞれの単語列をサーチエンジンで検索して、検索結果を表すページから出現頻度などを抽出しています。Settingsウィンドウの表は、使用するサーチエンジンのurl や、そのサーチエンジンが検索を行う際に使用するCGIのコマンドなどを設定します。各行が、それぞれのサーチエンジンの設定を表します。Google の場合は、Java で書かれたGoogleDriver を使用します。

 

       search-engine 欄は、サーチエンジンの名前を表します。

       driver 欄は、特別なプログラムを使用する場合に、そのプログラム名を記入します。

       url 欄は、サーチエンジンの URL を表します。

       query-head 欄は、単語列をサーチエンジンで検索するとき、CGI に与えるコマンドの先頭部分を記入します。

       word-separator は、単語列の、単語間の区切り記号を記入します。

       query-tail は、CGIに与えるコマンドの最後の部分を記入します。

       frequency-format は、検索したページの中で、出現頻度を表しているパターンを記述します。例えば、"found * results" は、検索した結果のページに、found <出現頻度> results が表れる場合、*　が出現頻度にマッチし、その値が出現頻度として抽出されることを表します。

       answer-list-format は、検索結果のページから、単語列を含む文の例やその単語列を含むページのURL を表すリストのパターンを記入します。(未完成)

       info-position は、answer-list-format に合致したパターンの中で、KWICページに表示する情報の位置(左から数えた*の順番)を表します。(未完成)

 

       以下は、webleap.cfg の内容の例です。

 

(((proxy ("false" "www.tobata.isc.kyutech.ac.jp" "8080" )

 )

 (config (((search-engine "Altavista" )

 (driver "" )

 (url "http://www.altavista.com/" )

 (query-head "web/results?q=%22" )

 (word-separator "+" )

 (query-tail "%22&kgs=0&kls=0&avkw=qtrp" )

 (frequency-format "found * results" )

 (answer-list-format "</b> * <br> * * * * </span>" )

 (info-position "5,3,3,0" )

 (url-pattern "URL:*" )

 (next-list "" )

 (previous-list "" )

 (comments "" )

 )
