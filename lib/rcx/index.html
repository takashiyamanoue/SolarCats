RCX and Java 
A  (100% JavaTM) programming interface to the Lego Mindstorms RCX
For general information on Lego Mindstorms RCX and the related mailing list see http://www.crynwr.com/lego-robotics

RCX Java API: API used to communicate with the RCX, used by RCXLoader.

RCXLoader: An example  Java application used to communicate with the Lego(tm) Mindstorms RCX brick, using the RCX Java API. Used as a tool to test the RCX protocol based on Kekoa Proudfoot's opcodes work at his RCX Internals site.

Last Updated: (10/22/99) - minor changes, updated documentation, licensing to LGPL!  Orignally released:(10/20/98)


Prerequisites:

A Java 1.1+ runtime environment as well as the Java Communications Extension: http://java.sun.com/products/javacomm (~270K)

Platforms currently supported are: Windows, Solaris Sparc, Solaris x86, Linux and Macintosh.

The MRJ (Macintosh Runtime for Java) is required for the Macintosh platform (serial port access through modem port). You can obtain the latest MRJ here: http://devworld.apple.com/java/text/prerelease.html as well as a 3rd party Java Comm implementation such as: http://www.vmeng.com/beard/javax.comm.MRJ/ Linux users require a Java Runtime as well as a 3rd party Java Comm implementation such as http://www.interstice.com/kevinh/linuxcomm.html


Download RCXLoader:

          rcx.zip (90K) - includes library and source files including the RCXLoader example as well as documentation.

To run: add comm.jar and rcx.zip to your classpath and specify rcx.RCXLoader as the main class to run.

Examples:

    jre -cp comm.jar;rcx.zip rcx.RCXLoader
    java -classpath [java lib path]classes.zip;comm.jar;rcx.zip RCXLoader

Note: Use jrew and javaw to run w/o a console. Omit comm.jar on Macintosh platforms (Java Comm API with Macintosh not available yet)

To use the Java Communication API: The only files you absolutely need are: javax.comm.properties and comm.jar (in the same folder).  The platform specific files for the Java Communcations API are:  for Windows: win32comm.dll (placed somewhere in your PATH)  and for Solaris: libSolarisSerialParallel.so (placed somewhere in your library path) For the other platforms consult the Java Communications documentation.
